Train Dynamics Calulator


The purpose of this program is to give insight into dynamics of braking annd acceleration for different trians. Currently this will be limited to just Victorian train but later may be applied more generally.

This should be used as a tool but its accuracy cannot be gauranteed as there are many limitations with the data, some calculations may be wrong and the work has not been checked or verified by another engineer other than myself.

Train Dynamics Data from VRIOGs carries the following limitations as stated in their standard

"The calculator provides the determination of speed, time and distance for various rolling stock. The acceleration and braking rates are based on information supplied by PTV / TfV / DEDJTR and the Accredited Rail Operator (ARO). The data source appears at the bottom of the screen when a calculation is performed. The calculations supported are acceleration, emergency braking, service braking or constant speed. The train types supported are Vlocity, Sprinter, N Class Passenger train, HCMT (restricted acceleration), X'Trapolis (restricted acceleration), Siemens Nexas (restricted acceleration), Short freight train up to 680m (GW16), medium length freight train up to 1200m (GW30), and long freight train up to 1500m (GW40). HCMT, Comeng, X'Trapolis and Siemens Nexas acceleration is based on the traction acceleration models produced by TfV / DEDJTR. HCMT braking is based on the HCMT specification. The calculator supports the affect of gradient. The gradient value entered will be applied along the distance of the calculation. The maximum/minimum gradient value supported is 3.3% for passenger trains or 1:40 for freight trains. For all passenger trains the gradient value is multipled by gravity and then added/subtracted to the train's base acceleration rate to form the effective rate. For freight trains the gradient adjustment is supplied directly from the GW16/GW30/GW40 tables. The acceleration and braking data used for the calculations is embedded in sheets of the workbook that are hidden and protected. To view these sheets select the 'Show All Sheets' button on the calculator. The calculator is available to be used for signal headway modelling but the following limitation and application considerations apply:

  • The data in the calculator is based on or derived from information supplied by PTV/TfV, DEDJTR and the ARO. The data is considered to be the best available information. The user should treat the results from the calculator as a guide. The actual data to be applied in a design is to be confirmed/advised by the ARO.
  • The braking data is based on stopping (i.e. a final velocity of 0 km/h) distance. This data has amortised the affect of the train brake pipe delay time. When the calculator is used for a braking calculation with a non zero final speed, the results will be understated (due to the affect of train brake delay). A margin should be added to any non zero final speed braking calculation.
  • When the calculator is used for a V/Line passenger train braking calculation (Vlocity, Sprinter, N Class) with an initial speed of 40 km/h or less, the results will be understated due to the affect of train brake delay. A margin should be added to the braking results. For HCMT and freight trains, the affect of train brake delay is already included in the stopping distance, so no additional margin needs to be considered.
  • The HCMT NFP acceleration is based on a restricted traction currents as will be used on the Cranbourne / Pakenham lines. Together with HCMT service braking HCMT this should be used for CPLU operational (headway) modelling."

Gradient Calulator


This is a tool that allows calculations for average gradients accross a profile of many track gradient section. It allows the user to enter a title, then enter as many gradient areas as wished. there is an option to add new tables if desired. This information can then be exported as a CSV or printed.

Basic Headway Calulator


This tool is designed to give a quick general idea of possible headways on a line given the train type and signalling technology. It takes into account assumptions but these assumptions can be tweaked in the general settings.

More trains will be added and also the theoretical scenarios of using ETCS or CBTC with these trains will eventually be implemented. The user interface also needs tweaking. It is a work in progress.

Time-Distance Graph Generator


This tool takes into account the train type, the braking and acceleration curves as well as the station stops. It outputs the time distance graph for the give journey.

This app uses graphs the braking and acceleration curves of each train by inputting piecewise equations into the Desmos API to graph the results.
